## YOLO Model Configuration
- **Model:** `backend/models/best.pt` (custom trained YOLO segmentation — ONLY model used)
- **NO fallback:** System requires best.pt — raises error if missing
- **Task:** `segment` (segmentation model)
- **Classes:** `{0: 'urban_land', 1: 'agriculture', 2: 'barren_land'}` — 3 land-use classes
- **Model loading:** Singleton pattern, pre-loaded on server startup
- **Confidence threshold:** 0.25 (segmentation-optimized)
- **Output:** Segmentation masks + bounding boxes per detection

## Analytics Engine
Heuristic land-use intelligence from 3-class segmentation (urban_land → building/urban/infra metrics, agriculture → vegetation, barren_land → degradation/stress):
- **building_density** — derived from urban_land area (× 0.85 factor)
- **vegetation_coverage** — agriculture mask coverage %
- **urban_density** — urban_land mask coverage %
- **water_presence** — estimated from vegetation context
- **environmental_stress** — low vegetation + high urban + barren exposure
- **infrastructure_concentration** — derived from urban_land (× 0.7 factor)
- **risk_score** — composite weighted (0–100)
- **risk_level** — LOW / MEDIUM / HIGH / CRITICAL

## AI Copilot Context
Detection-grounded insights generated from actual best.pt output:
- No hallucination — all metrics reference real segmentation data
- Land-use specific insights (encroachment, drainage, deforestation)
- Actionable recommendations tied to risk levels
- Drainage congestion heuristic (high buildings + low vegetation + water)
